# v0.0.0 — The Foundation

Ships myfi as a complete, installable Claude Code finance plugin: a six-agent flock, four skills, four
entry-point commands, a Python3/poetry toolkit (CLI + stdio MCP) carrying numpy/pandas/scipy/matplotlib,
per-project + optional global SQLite, a market-data adapter contract, local-Claude-Code LLM + eval
services, ported harness discipline, and a real end-to-end financial report via `/myfi:analyze`.

**The seed is ground truth.** This plan decomposes it; it does not rescope it. All 16 §6 deliverables map
1:1 to 16 units. See `## Coverage map`.

## Topology convention (READ FIRST — binds every unit)

- Root executes waves **sequentially**. Within a wave, units fan out **concurrently** as `shepherd:coder`
  subagents, each gated by a `shepherd:auditor` (wave-review mode, PASS/REDO, cap 3), then root commits the
  wave. Cross-wave deps are fine (later waves depend on earlier); **same-wave units are file-disjoint.**
- **Disjointness is at FILE granularity with enumerated ownership.** Two same-wave units NEVER write the
  same file. Where units share a parent dir (`agents/`, `services/toolkit/tests/`, `services/eval/rubrics/`),
  each owns an **enumerated, mutually-exclusive file set** and creates ONLY its own files. This is exactly
  the seed's "file-disjoint lanes" and "ADD files to already-relocated dirs" model — the Wave 1 relocation
  creates the parent dirs so later units only add into them.
- **The relocation (Wave 1) is solo and gates everything.** No later unit writes into `agents/`, `skills/`,
  `commands/`, `bin/`, or `.mcp.json` until Wave 1 has moved `src/*` to root and committed.
- **`<2s` fast-gate rule:** any code the fast gate imports (CLI `--version`, module import smoke) MUST NOT
  top-level-import numpy / pandas / scipy / matplotlib / mcp. Heavy libs are **lazy-imported inside the
  handler that needs them**; matplotlib uses the **Agg** backend set before any pyplot import.
- **LLM law (CLAUDE.md, non-negotiable):** every model call routes through `services/llm/llm.py` shelling to
  local Claude Code. `rg -n 'api\.(anthropic|openai)\.com' services/ commands/ agents/ skills/` MUST return 0.
- **Tests + evals same commit (CLAUDE.md):** every unit ships gate tests (deterministic, local, <2s) AND an
  eval (a `services/eval` rubric + golden good/bad fixtures, scored by local Claude Code on the live lane,
  mock-lane-asserted on the gate lane). Deterministic-only units (relocation, db, hooks) whose behavior has
  no latent surface ship a **golden/round-trip eval** instead and say so explicitly — that is the honest
  application of the latent/deterministic split, not an eval skip.

## Shared naming (locked — every unit uses these)

- Package `myfi_toolkit`; subpackages `myfi_toolkit.myctx` (DB), `myfi_toolkit.marketdata`.
- Console scripts: `myfi-toolkit = myfi_toolkit.cli:main`, `myfi-mcp = myfi_toolkit.mcp_server:main`.
- Root wrappers `bin/myfi-toolkit`, `bin/myfi-mcp`: `poetry -C "$ROOT/services/toolkit" run <script> "$@"`
  with `python -m myfi_toolkit.<mod>` fallback when poetry is absent.
- MCP registered in root `.mcp.json` as server `myfi-toolkit` → command `${CLAUDE_PLUGIN_ROOT}/bin/myfi-mcp`.
  Tools surface as `mcp__plugin_myfi_myfi-toolkit__<tool>`.
- Env (mirror shepherd, `MYFI_` prefix): `MYFI_LLM_BIN` (default `claude`), `MYFI_LLM_MODEL` (default `opus`),

- DBs: per-project `.myfi/myfi.db` (git-ignored — already in `.gitignore`), optional `~/.myfi/global.db`.
- Test lanes: **toolkit** = `poetry -C services/toolkit run pytest -q` (the `[gates].check`); **stdlib
  services** (`services/llm`, `services/eval`) = `python3 -m unittest discover` (no deps, mock seam, <2s);
  **hooks** = `bash hooks/tests/run.sh`. Eval rubrics live in `services/eval/rubrics/*.rubric.json`.
