# v0.0.0 Close -- The Foundation

## Verdict: DONE_WITH_CONCERNS

myfi v0.0.0 ships as a complete, installable Claude Code finance plugin. Every one of the seed's
16 issue-anchored deliverables landed with gate tests AND evals in the same commit. The full
release gate (`bin/myfi-test`) is green across all four suites, the plugin auto-discovers every
component, and `/myfi:analyze` produces a real financial report artifact end to end. The concerns
are deferred follow-ups the seed itself anticipated (nightly live-eval lanes, T+1d/T+7d soak) plus
two optional enhancements -- none is incomplete sprint work.

## Adversarial audit yield (defects the auditor caught, root fixed)

Every wave was gated by a `shepherd:auditor` running the unit's real acceptance predicate. Real
defects it surfaced and root resolved (all committed, all logged in the plan's deviation table):

1. **MCP-tools grounding gap (HIGH, cross-wave)** -- `SKILL.md` + all six agents cited 4 MCP tools
   (`quote`, `db_init/migrate/version`) that only existed as CLI verbs; `@advisor` (no Bash by
   design) had no working data path. Fix: registered them as real MCP tools wrapping the same
   library functions (`6553d0e`). Now all agent tool refs resolve; MCP lists 5 tools.
2. **Baseline checksum placeholder (HIGH)** -- `0001_init.sql` self-inserted a literal checksum,
   shadowing the real sha256 `migrate.py` computes. Fix: dropped the self-insert + regression test
   (`5e99ef9`); verified fresh-db checksum == sha256(file).
3. **Plan-text `poetry -C` double-prefix (recurring)** -- acceptance lines double-prefixed pytest
   paths (`poetry -C` chdirs first). Fixed plan-wide.
4. **Test/eval files outside a unit's file_scope** (W2-myfi-skill, W7-docs) -- coders correctly
   refused; root authored the gate artifacts + widened scope.

The adversarial pairing worked as the seed intended: coders built, auditors tried to prove corners
were cut, and the load-bearing MCP gap (which no acceptance predicate checked) was caught by an
auditor reasoning beyond its checklist.

## Restart / activation

The plugin is markdown + a poetry toolkit; nothing is running that needs a restart. To activate:

1. `cd services/toolkit && poetry install` (first time -- resolves numpy/pandas/scipy/matplotlib/mcp;
   already done this session, `poetry.lock` committed).
2. Reload the plugin so Claude Code re-discovers components + the MCP server: `/reload-plugins` in an
   interactive Claude Code session (the `.mcp.json` server `myfi-toolkit` starts on next session).
3. Verify: `bin/myfi-test` (4/4 suites) and `bin/myfi-toolkit --version`.
4. Try it: `/myfi:analyze <subject>` for a report, `/myfi:plan <goal>` for the flock pipeline.
